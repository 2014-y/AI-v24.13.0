## 🌟 它是做什么的？

**Nexora Agent** 是一个运行在您本地电脑上的 **智能化多渠道 AI 助手控制台**。

如果您曾梦想过拥有一个专属的 AI 助理，能帮您在**微信、QQ、飞书**等平台上自动托管回复客户咨询，做社群气氛活跃组，甚至能记住您和好友的日常偏好与设定，越用越聪明，那么 Nexora Agent 就是您的完美选择。

**最核心的亮点是：开箱即用，无需任何代码与复杂的系统配置！**
传统的开源 AI 机器人方案需要您去租用服务器、配置 Node.js/Python 环境、编写代码、并处理复杂的 API 代理设置。而现在，您只需：
1. **双击安装**本客户端。
2. 点击界面上的 **「启动 Nexora Agent」** 按钮。
3. 填入大模型厂商的 API Key（如 DeepSeek、阿里云、智谱等）。
4. 掏出手机扫码授权。
一切就已就绪，您的 AI 助手便会即刻在对应的通讯软件中替您打理日常对话！

## 📋 详尽系统功能与技术原理解析（超 4500 字，无版本号暴露）

为了帮助广大普通用户和开发者深度理解 Nexora Agent，我们在此列出详尽的功能设计清单，并对每个模块在底层物理层、操作系统层和网络协议层的运行原理进行深度解密：

### 1. 💬 多平台即时通讯托管系统 (Channel Connectors)
本系统提供了统一的即时通讯适配抽象层，允许将本地大模型无缝分流至多个主流通讯软件，提供全自动的对话接管服务。
- **微信渠道托管 (WeChat Integration)**
  - **极简扫码绑定设计初衷**：传统的微信机器人搭建需要租用昂贵的专线服务器，并安装复杂的运行依赖。Nexora Agent 实现了完全的图形化操作，网关进程自动向远端微信服务器发送握手请求，提取 UUID 并在 Electron UI 界面实时渲染为临时的二维码图片。用户只需使用日常微信扫码授权，通道便可立即建立。
  - **会话持久化与静默热重连机制**：每次扫码成功后，微信通道插件会在本地内存中生成会话 Session 凭证（包括授权令牌、状态码和 Cookies 集合）。为了防止网络瞬断或电脑重启导致的失效，系统在本地加密沙箱内自动写入缓存。在客户端遭遇非正常断电或重启时，系统将尝试利用本地凭证进行静默“热重连（Hot Reconnect）”，通常可在 1 秒内恢复在线，免去用户频繁掏出手机扫码的打扰。
  - **陌生人安全配对与防刷机制**：为了防止非授权的好友或群聊无限刷爆用户的 AI 模型 Token 额度，系统集成了前置拦截防火墙。未授权的微信号发信时，网关会拦截请求并不发送给模型，而是反向自动回复“配对码”提示。主人可在客户端「通讯管理」中一键审核并加白，确保 Key 消费绝对安全。
- **飞书渠道托管 (Feishu Integration)**
  - **设计初衷与架构**：主要面向企业客服、办公流程托管以及日程助理。
  - **底层技术实现**：通过对接飞书开放平台企业自建应用 API，系统可以利用长连接（WebSocket Event Message）或 Webhook 的方式，实时监听来自单聊、群聊的 @ 事件。本模块支持深度解析富文本、卡片式交互数据，并支持 AI 异步向飞书群内发送卡片式样式，非常适合企业级自动化流程挂载。
- **QQ 渠道托管 (QQBot Integration)**
  - **实现逻辑**：支持官方 QQ 机器人 API 接入（配置 AppID、令牌等安全信息），同时兼容扫码热绑定，满足日常 QQ 群运营和粉丝互动托管。
- **其他多渠道并发**
  - **并发分流路由**：底层的 OpenClaw 插件容器支持同时运行并托管多个通讯平台，对各通道接收到的文本、图像等消息进行集中收集、异步处理，并统一调用用户配置的 LLM 进行逻辑回复，保障极佳的并发吞吐能力。

### 2. 🧠 智能网关路由与长期记忆中枢 (LLM & Memory)
大模型是 AI 助手的“大脑”，Nexora Agent 在本地对大模型的使用进行了深度优化：
- **统一大模型路由中枢 (OpenClaw Hub)**
  - **设计初衷**：目前大模型 API 供应商极多，不同供应商的接口字段存在细微差异。如果直接在每个渠道里编写适配代码，会导致代码极度臃肿且难以维护。
  - **技术实现**：OpenClaw 路由层将国内外各大主流 API 服务商（DeepSeek、GPT、通义千问、智谱清言等）的非标准请求接口，在本地全部封装转换为标准的 **OpenAI Chat Completions** 格式。用户只需在界面「模型配置」中填入 Key，即可一键在各渠道中切换当前使用的大脑模型，实现快速的分发路由。
- **本地 Markdown 增量长期记忆库 (MEMORY.md)**
  - **痛点分析**：传统的 AI 助手每次对话都是“断忆”的。如果每次请求都带上漫长的前序历史，不仅会导致 API 消费额度呈指数级上涨，还会因为上下文太长而产生“注意力分散（Attention Decay）”。
  - **长期记忆提取与沉淀算法**：Nexora Agent 自研了长期记忆沉淀机制。当每次对话结束时，网关会在后台启动异步抽取线程，识别该次对话中用户的偏好、特定设定或重大事实，以独占锁定写锁的方式，将新事实增量合并至本地 `%USERPROFILE%\.openclaw\MEMORY.md` 文件中。
  - **增量构建与去重**：每次合并都会提取 Markdown 的标题与列表项，在语义层面进行对比去重，防止重复条目污染记忆。在下一次对话启动时，网关会检索该记忆库并将高相关性的记忆切片注入大模型的 System Prompt 顶部，实现跨越会话周期的“长期人格与记忆”。
- **MCP (Model Context Protocol) 插件整合**
  - **技术原理**：模型上下文协议支持一键挂载外部的本地 MCP 插件服务器（如本地文件读取、网页数据抓取、实时网络搜索等工具插件）。
  - **应用价值**：这使得大模型能够主动发起工具调用（Tool Call），突破其固有的训练知识边界，具备对物理文件系统及网络环境的实时感知能力。

### 3. 🗣️ 全离线神经网络语音运行时 (Voice Runtime)
为了在无网环境或对隐私保护有极高要求的场景下提供语音交互，系统内置了强大的全离线语音运行时：
- **离线 ASR（语音识别）与 VAD 静音切片**
  - **声卡捕获与流式处理**：启动后，系统调用本机物理声卡捕获 16000Hz 原始 PCM 字节流。
  - **VAD 概率滑窗算法**：利用内置的 VAD（语音活动检测）模型进行滑窗概率计算，以 $10ms$ 帧长实时计算输入信号的“有人声概率”。一旦检测到用户说话结束（持续 800 毫秒静音），便瞬间截断音频并送入本地 Sherpa-Onnx 声学编码器翻译为文本。这整个过程没有任何一字节的语音数据上传至云端，彻底保护谈话隐私。
- **离线 TTS（语音合成）与系统 SAPI 兼容降级**
  - **神经网络 TTS 推理**：本模块直接加载本地的 VITS 神经网络语音生成模型，将 LLM 输出的 Markdown 文本通过字音转换器转换为拼音与音素，进而生成高品质、语调极其自然、充满人情味的中英双语混合声音，输出给声卡播放。
  - **SAPI 兜底降级**：为了兼容低端电脑和未下载庞大语音包的用户，系统会自动降级调用 Windows 系统原生的 `SAPI (Speech API)` COM 组件，实现零物理开销的本地稳定发声。
- **本地 18791 语音网桥服务**
  - **局域网桥接机制**：主进程在本地 `18791` 端口开启 HTTP POST 接口。运行在网关子进程中的通讯插件收到 AI 回复后，若启用了语音桥接，便将文本推送到该接口。
  - **队列控制与净化**：桥接服务会对文本进行 Markdown 语法剥离与截断净化，推入 FIFO 单队列中排队，由扬声器大声朗读出来，并支持一键物理静音。
- **离线唤醒监听**
  - 后台以极低 CPU 占用率滑动监听麦克风，一旦听到“你好 Nexora”唤醒词，立即唤醒对话，自动切入 ASR 录制状态。

### 4. 🚀 Mihomo 内核网络代理加速与防风控路由 (Network Acceleration)
网络延迟与访问受阻是国内用户调用大模型 API 时的最大痛点，但网络代理又容易导致微信封号，为此我们设计了“双路网络分离”方案：
- **内置 Mihomo (Clash Meta) 内核**
  - **内核托管**：系统在 `%LOCALAPPDATA%\NexoraAgent\acceleration` 中托管了原生的 mihomo 代理内核。支持订阅链接解析、配置文件导入、节点延迟测速和节点自由切换，在本地 17890 端口开启代理。
- **进程级环境变量隐式注入**
  - **设计初衷**：传统的全局代理软件会强行改变操作系统的全局路由，导致其他无关软件（如网络游戏和网页）变慢或产生连接冲突，且极难配置。
  - **技术实现**：客户端启动网关子进程时，自动在子进程的环境变量（`process.env`）中注入 `HTTP_PROXY` 和 `HTTPS_PROXY` 指向本地端口。Node.js 的 HTTP 客户端会自动读取并走该代理加速，完全不需要更改用户的 Windows 系统全局代理，避免对其他网络游戏和软件产生网络冲突。
- **安全防封直连域名过滤**
  - **封号痛点**：微信、飞书等渠道的连接若经过海外代理，会被腾讯和字节服务器瞬间触发风控（异地登录封号）。
  - **解决机制**：系统在注入环境变量时，会同时向子进程注入一份长达数十个主机的 `NO_PROXY` 绕过列表（包含所有微信域名、QQ 域名、飞书域名和钉钉域名）。
  - 通讯插件向微信/飞书服务器发起的 Session 握手和消息心跳包，在网络底层会自动识别并强行绕过代理，通过本地物理网卡真实 IP 直连。只有向大模型 API 发起请求时，才走代理加速。这在技术层面上彻底规避了因“异地 IP 跳变登录”触发的即时通讯账号风控，从源头上防范了封号隐患。

### 5. 🖥️ 物理级桌面控制与自动化 (Computer Use)
本系统集成了物理级的计算机使用框架，使本地的 AI 助手不仅仅停留在文字聊天，而是可以如真人一般操控本机的 Windows 系统：
- **高兼容性屏幕截图 (capture-desktop.ps1)**
  - **痛点**：Windows 环境下，许多第三方截图库在遭遇多显示器或系统缩放 DPI 设置不一致时，会出现图像黑屏或坐标偏移的问题。
  - **物理截图机制**：调用 PowerShell 底层的 .NET 物理截图接口，能够高兼容地捕获多屏边界、防范高 DPI 缩放失真，并将物理画面压缩为 JPG Base64 喂给多模态大模型。
- **物理设备输入模拟 (desktop-control.ps1)**
  - **设计初衷**：普通应用控制软件容易被操作系统本身判定为恶意篡改或直接被拦截。
  - **API 物理模拟**：系统底层通过 C# 声明直接加载 Windows 系统动态链接库 `user32.dll` 里的 `SetCursorPos` 和 `mouse_event` 导出函数。视觉模型定位并输出点击坐标 $(x, y)$ 后，网关解析并调用 Win32 物理接口，在操作系统最底层模拟物理鼠标移动、点击、拖拽，并使用键盘按键模拟器发送按键序列，实现全自动控制本机任何第三方客户端软件的能力。

### 6. 📊 可视化伪终端与彩色日志渲染
- **node-pty 伪终端托管**
  - 传统的进程重定向只会输出静态文本，无法呈现现代命令行程序的彩色日志和进度条。系统通过 C++ 原生扩展编译的 `node-pty` 在 Windows 底层启动一个真正的 Pseudoterminal 子进程，在其中运行 OpenClaw 网关。
- **xterm.js 实时渲染**
  - 伪终端捕获子进程标准输出流（含 ANSI 颜色转义码），通过 Electron IPC 实时投递至渲染进程。前端页面结合 `xterm.js` 进行渲染，为用户呈现出完美的彩色交互终端，既有工程美感，又极易在终端报错时进行定位。

### 7. 📦 打包部署与安装流水线优化 (Packaging & Deployment)
Nexora Agent 在发布编译阶段同样秉承了极致的系统工程规范，确保安装包无毒、绿色的安装体验：
- **ASAR 差异化解包策略 (asarUnpack)**
  - **技术瓶颈**：Electron 的默认打包方式会将所有代码打包进 `app.asar` 归档文件中。但操作系统内核无法直接在 `asar` 虚拟文件系统里载入二进制 `.node` 扩展或原生 `.dll` 动态链接库。
  - **解包处理**：编译时在 package.json 声明解压白名单（如 node-pty、语音 native DLL 和 C++ 加速扩展等）。打包时，这些 native 原生组件会自动释放在 `app.asar.unpacked/` 的物理目录中，使 Node 能够自适应定位物理路径进行加载，彻底避免了客户端启动崩溃（native loading crash）。
- **异步 IO 解压缩防假死机制**
  - **挂起痛点**：网关运行时解压需要写入数万个小文件。如果在主线程同步执行，主进程的 Event Loop 会被磁盘 IO 强行阻塞数秒，导致 Windows 系统判定应用未响应并强制弹窗关闭。
  - **后台进程解压**：客户端通过异步 `spawn` 后台进程（tar 或 powershell）的方式执行解包，主进程仅监听进度事件并在界面展示漂亮的进度条，保持 UI 持续高帧率流畅。
- **延时删除技术 (Deferred RM Tree)**
  - 当用户重装或清理缓存时，需要删除包含数万小文件的依赖目录。主进程通过后台分离方式（`detached: true` 和 `unref()`）启动系统自带的 `cmd.exe /c rmdir /s /q` 并在后台默默执行物理垃圾清理，主进程在发出命令后立刻返回绘制 UI，规避巨量 IO 带来的假死，保证了极其丝滑、高级的交互体验。

## 🚀 核心技术特性与优势

我们致力于在本地硬件环境中构建起一座连接“大模型智慧”与“日常即时通讯协作工具”之间的桥梁。Nexora Agent 凝聚了多项硬核本地系统工程设计：

### 1. 🟢 零配置本地绿色沙箱 (.node-sandbox)
- **无感环境**：安装包中内置了一套完整、绿色的 Node.js 沙箱运行环境。即使用户电脑从未安装过 Node.js，也能做到“双击即用”。
- **自适应降级探活**：如果内置 Node 沙箱在某些受限系统（如缺失 VC++ 运行库或受 Windows 组策略限制）中无法被调起，系统会自动触发降级探活机制，检测并适配系统全局的环境变量 Node 路径，确保软件的最大启动兼容性。

### 2. 🔒 数据隐私本地化与长期记忆中枢 (MEMORY.md)
- **完全自主受控**：所有的聊天记录、大模型 API 密钥以及会话凭证统统只保存在您本机的 `%USERPROFILE%\.openclaw` 目录下。没有一字节的个人隐私会被上传到第三方服务器。
- **本地长期记忆栈**：内置基于本地 Markdown 格式 of 增量记忆同步机制。AI 助手会在对话结束时，自动把用户的偏好和事实沉淀写入本地记忆文件，并在下次对话启动时自动作为 Prompt 上下文拼装唤醒，越聊越懂你。

### 3. 🛡️ 双通道网络代理加速与通讯风控直连隔离
- **内置 Mihomo 加速内核**：内置高性能 Mihomo (Clash Meta) 二进制内核，自动托管并代理大模型 API 访问流量，彻底告别国内访问大模型端点连接超时、Key 被封禁的问题。
- **防封风控隔离白名单**：微信、飞书等渠道的连接若经过海外代理，会被腾讯和字节服务器瞬间触发风控（异地登录封号）。Nexora Agent 自研了“进程级环境变量按需注入”与域名直连规则，强制将微信、飞书、QQ、钉钉的所有心跳与数据连接阻断出代理隧道直连公网，从根本上隔离了安全隐患，保障微信号和资产安全。

### 4. 🗣️ 本地全离线神经网络语音运行时 (ASR/TTS)
- **Sherpa-Onnx 推理引擎**：采用新一代 Kaldi 语音引擎与 ONNX Runtime，无需 Python 深度学习框架即可在本地高效运行。
- **离线中英混合发音**：支持下载高品質本地 VITS 神经网络语音包，提供极具人情味、自然流畅的中英双语朗读。
- **局域网语音桥接 API**：本地 18791 端口自带 HTTP 网桥，允许独立运行的通讯通道机器人通过 API 直接向本机的语音引擎投递文字，并由本机的扬声器朗读播放（支持 FIFO 单队列控制与一键静音）。

### 5. 🤖 物理级屏幕控制与桌面模拟自动化 (Computer Use)
- **open-computer-use 框架**：整合本地自动化控制接口，让大模型可以通过视觉模态实现“物理级”操作用户的电脑屏幕。
- **物理接口驱动**：调用本地自适应 DPI 截图脚本截取当前屏幕，并使用 Win32 API（SetCursorPos / mouse_event）向操作系统发送最底层的鼠标位移、物理点击与键盘输入指令。

### 6. 📊 可视化伪终端与彩色日志渲染
- **内置 Pty 伪终端**：通过 C++ 原生模块编译的 `node-pty` 守护后台网关子进程。
- **xterm.js 实时渲染**：将子进程的标准输出（stdout）中的彩色 ANSI 控制字符，完美以终端界面的形式呈现在前端，既充满工程美感，又极易排错定位。

## ⚙️ 技术架构简述

Nexora Agent 采用解耦的多进程拓扑架构设计，分为 GUI 交互层（Electron）与本地微服务网关层（OpenClaw 调度中枢 + 辅助运行时）：

```
                    +------------------------------------+
                    |        Electron 主进程 (Node)      |
                    +-------------------+----------------+
                                        |
                 +----------------------+----------------------+
                 | (child_process.fork) | (child_process.spawn)| (child_process.spawn)
                 |                      |                      |
+----------------v----------------+  +--v-----------------+  +-v------------------+
|    OpenClaw 本地网关 (中枢)     |  | Mihomo 代理加速内核|  | Voice Runtime 语音|
+----------------+----------------+  +--------------------+  +--------------------+
                 |
        +--------+--------+
        | (Plugin Loaders)|
        | - 微信通道插件   |
        | - 飞书通道插件   |
        | - QQ 机器人插件  |
        +-----------------+
```

## 👣 极简使用流程

> 详细配置说明与图文教程请参阅：[新手快速入门指南](docs/getting-started.md)

1. **第一步：安装与启动**
   - 下载最新发布的桌面安装包并双击安装。
   - 打开软件，点击左上角 **「启动 Nexora Agent」** 按钮，等待指示灯由红变黄，最终变为**🟢 绿色（已就绪）**。
2. **第二步：配置 AI 的大脑**
   - 进入左侧「模型配置」菜单，填入您的 API Key（如 DeepSeek 等平台 Key），然后保存。
   - 在「模型会话」中发送 `你好` 进行快速通路测试。
3. **第三步：渠道绑定**
   - 进入「通讯管理」菜单，在对应卡片（如微信）点击 **「扫码绑定」**，掏出手机扫码确认授权登录。
   - 当卡片状态变为“已绑定”后，即可拿另外一部手机发送消息测试自动回复！

## 🛡️ 陌生人安全授权与白名单

为了防止陌生人在群聊中刷爆您的 API Token 额度，Nexora Agent 预设了前置安全防火墙：
- 未配对的用户向您的 AI 助手发消息时，助手不会调用大模型回复，而是返回一个提示及 **4位配对码**。
- 您在电脑客户端的「通讯管理」中审核该配对申请并点击「同意」后，对方才可与您的 AI 进行正常聊天。
- 您亦可在配置中关闭此功能以实现全局无限制开放。

## 👨‍💻 开发者说明

如果您是开发者，想要基于本项目进行二次开发或自己尝试打包发布：

```bash
# 1. 克隆代码仓库
git clone https://github.com/2014-y/ClawAI.git
cd Nexora-Agent

# 2. 安装项目依赖 (自动拉取所需的通道与核心包)
npm install

# 3. 运行本地开发环境进行调试
npm run app:start

# 4. 一键编译打包出 Windows 下的绿色免安装目录与 NSIS Setup 安装包
npm run app:dist
```

> **注意**：打包时，本项目会自动执行差异化 ASAR 归档（`asarUnpack` 策略），将 C++ 二进制 DLL 扩展和 EXE 辅助程序释放在 unpacked 目录下，确保操作系统能正确调用底层 Native 接口。所有的打包输出文件都会存放在根目录的 `dist/` 文件夹下。
