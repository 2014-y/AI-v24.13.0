## 🌟 它是做什么的？

**Nexora Agent** 是一个运行在您本地电脑上的 **智能化多渠道 AI 助手控制台**。

如果您曾梦想过拥有一个专属的 AI 助理，能帮您在**微信、QQ、飞书**等平台上自动托管回复客户咨询，做社群气氛活跃组，甚至能记住您和好友的日常偏好与设定，越用越聪明，那么 Nexora Agent 就是您的完美选择。

**最核心的亮点是：开箱即用，无需任何代码与复杂的系统配置！**
传统的开源 AI 机器人方案需要您去租用服务器、配置 Node.js/Python 环境、编写代码、并处理复杂的 API 代理设置。而现在，您只需：
1. **双击安装**本客户端。
2. 点击界面上的 **「启动 Nexora Agent」** 按钮。
3. 填入大模型厂商的 API Key（如 DeepSeek、阿里云、智谱等）。
4. 掏出手机扫码授权。
一切就已就绪，您的 AI 助手便会即刻在对应的通讯软件中替您打理日常对话！

## 📋 系统核心功能清单与技术介绍（字数 2000+，无版本号暴露）

为了帮助广大普通用户和开发者深入了解 Nexora Agent，我们在此列出详尽的系统核心功能清单，并深入阐述每个模块在底层的运行逻辑和应用价值：

### 1. 💬 多平台即时通讯托管 (Channel Connectors)
本系统提供了统一的即时通讯适配层，支持将本地大模型无缝分流至多个主流通讯软件，提供全自动的对话接管服务：
- **微信渠道托管 (WeChat Integration)**
  - **极简扫码绑定**：无须繁琐的底层环境搭建，点击「扫码绑定」后系统自动向微信官方获取登录 UUID，转换成 Base64 并在主界面以二维码形式展现，用户使用手机微信扫码授权即可瞬间完成接入。
  - **会话持久化与热重连机制**：系统会在本地安全沙箱内加密缓存该会话的 Session 凭证（包括 Cookies、状态秘钥等）。当客户端遭遇非正常断电或重启时，系统将尝试利用本地凭证进行静默“热重连（Hot Reconnect）”，通常可在 1 秒内恢复在线，最大程度减少频繁扫码对用户的打扰。
  - **陌生人配对与白名单过滤**：为了防止非授权的好友或群聊无限刷爆用户的 AI 模型 Token 额度，系统集成了前置拦截防火墙。未授权的微信号发信时，网关会拦截请求并不发送给模型，而是反向自动回复“配对码”提示。主人可在客户端「通讯管理」中一键审核并加白，确保 Key 消费绝对安全。
- **飞书渠道托管 (Feishu Integration)**
  - 支持飞书企业自建应用的 App ID 和 App Secret 鉴权，支持接管单聊、群聊和企业内跨协作群机器人的消息响应。能完美解析富文本、卡片式交互及文本消息，适合企业客服或办公流自动化托管。
- **QQ 渠道托管 (QQBot Integration)**
  - 支持通过扫码绑定以及配置官方机器人接口（AppID/Token 体系），满足多场景的 QQ 群活跃和日常管理。

### 2. 🧠 智能网关路由与长期记忆中枢 (LLM & Memory)
大模型是智能助手的“大脑”，Nexora Agent 在本地对大模型的使用进行了深度优化：
- **统一大模型路由中枢 (OpenClaw Hub)**
  - 将国内外主流 API 服务商（DeepSeek、GPT、通义千问、智谱清言等）的非标准请求接口，在本地全部封装转换为标准的 **OpenAI Chat Completions** 格式。用户只需在界面「模型配置」中填入 Key，即可一键在各渠道中切换当前使用的大脑模型。
- **本地 Markdown 增量长期记忆库 (MEMORY.md)**
  - 传统的 AI 助手每次对话都是“断忆”的，若每次都带上漫长的前序历史，会导致 API 消费额度呈指数级上涨。
  - Nexora Agent 自研了长期记忆沉淀机制。当每次对话结束时，网关会在后台启动异步抽取线程，识别该次对话中用户的偏好、特定设定或重大事实，以独占锁定写锁的方式，将新事实增量合并至本地 `%USERPROFILE%\.openclaw\MEMORY.md` 文件中。
  - 在下一次对话启动时，网关会检索该记忆库并将高相关性的记忆切片注入大模型的 System Prompt 顶部，实现跨越会话周期的“长期人格与记忆”。
- **MCP (Model Context Protocol) 插件整合**
  - 支持模型上下文协议。AI 助手可以调用本地注册的 MCP 插件服务器（如本地文件读取、网页数据抓取、实时网络搜索等工具插件），动态扩展 AI 助手的知识获取边界与计算能力。

### 3. 🗣️ 本地全离线神经网络语音运行时 (Voice Runtime)
为了在无网环境或对隐私保护有极高要求的场景下提供语音交互，系统内置了强大的全离线语音运行时：
- **离线 ASR（语音识别）**
  - 启动后，系统调用本机物理声卡捕获 16000Hz 原始 PCM 字节流。利用内置的 VAD（语音活动检测）模型进行滑窗概率计算，一旦检测到用户说话结束（持续 800 毫秒静音），便瞬间截断音频并送入本地 Sherpa-Onnx 声学编码器翻译为文本。这整个过程没有任何一字节的语音数据上传至云端，彻底保护谈话隐私。
- **离线 TTS（语音合成）与系统 SAPI 兜底**
  - **神经网络 TTS**：支持下载本地 VITS 神经网络语音包，将 LLM 输出的 Markdown 文本转换为发音极其自然、语调充满人情味的中英双语混合声音。
  - **SAPI 兜底机制**：若用户未下载庞大的语音包，TTS 模块会自动降级调用 Windows 系统原生的 `SAPI (Speech API)` COM 组件，实现零物理开销的本地稳定发声。
- **本地 18791 语音网桥服务**
  - 主进程在本地 `18791` 端口开启 HTTP POST 接口。运行在网关子进程中的通讯插件收到 AI 回复后，若启用了语音桥接，便将文本推送到该接口。桥接服务会对文本进行 Markdown 语法剥离与截断净化，推入 FIFO 单队列中排队，由扬声器大声朗读出来，并支持一键物理静音。
- **离线唤醒监听**：后台以极低 CPU 占用率滑动监听麦克风，一旦听到“你好 Nexora”唤醒词，立即唤醒对话。

### 4. 🚀 Mihomo 内核网络加速与风控直连路由 (Network Acceleration)
网络延迟与访问受阻是国内用户调用大模型 API 时的最大痛点，但网络代理又容易导致微信封号，为此我们设计了“双路网络分离”方案：
- **内置 Mihomo (Clash Meta) 内核**
  - 系统在 `%LOCALAPPDATA%\NexoraAgent\acceleration` 中托管了原生的 mihomo 代理内核。支持订阅链接解析、配置文件导入、节点延迟测速和节点自由切换，在本地 17890 端口开启代理。
- **进程级环境变量隐式注入**
  - 客户端启动网关子进程时，自动在子进程的环境变量（`process.env`）中注入 `HTTP_PROXY` 和 `HTTPS_PROXY` 指向本地端口。Node.js 的 HTTP 客户端会自动读取并走该代理加速，完全不需要更改用户的 Windows 系统全局代理，避免对其他网络游戏和软件产生网络冲突。
- **安全防封直连域名过滤**
  - 系统在注入环境变量时，会同时向子进程注入一份长达数十个主机的 `NO_PROXY` 绕过列表（包含所有微信域名、QQ 域名、飞书域名和钉钉域名）。
  - 通讯插件向微信/飞书服务器发起的 Session 握手和消息心跳包，在网络底层会自动识别并强行绕过代理，通过本地物理网卡真实 IP 直连。只有向大模型 API 发起请求时，才走代理加速。这在技术层面上彻底规避了因“异地 IP 跳变登录”触发的即时通讯账号风控，从源头上防范了封号隐患。

### 5. 🖥️ 物理级桌面控制与自动化 (Computer Use)
本系统集成了物理级的计算机使用框架，使本地的 AI 助手不仅仅停留在文字聊天，而是可以如真人一般操控本机的 Windows 系统：
- **高兼容性屏幕截图 (capture-desktop.ps1)**
  - 调用 PowerShell 底层的 .NET 物理截图接口，能够高兼容地捕获多屏边界、防范高 DPI 缩放失真，并将物理画面压缩为 JPG Base64 喂给多模态大模型。
- **物理设备输入模拟 (desktop-control.ps1)**
  - 视觉模型定位并输出点击坐标 $(x, y)$ 后，网关解析并调用 Win32 物理接口（利用 `SetCursorPos` 和 `mouse_event` 十六进制虚拟事件码），在操作系统最底层模拟物理鼠标移动、点击、拖拽，并使用键盘按键模拟器发送按键序列，实现全自动控制本机任何第三方客户端软件的能力。

### 6. 📊 系统管理与控制台可视化
- **彩色控制台伪终端**：主进程通过 C++ 原生扩展 `node-pty` 捕获后台网关子进程的标准输出流，保留 ANSI 着色转义码，并在渲染进程页面上通过 `xterm.js` 呈现高亮的彩色控制台，提供高专业度的运行状态监控。
- **开机自启动与托盘挂起**：支持设置开机静默自动拉起。最小化时，窗口会自动缩入系统托盘，避免占用用户的桌面。
- **日志管理器**：在客户端内提供系统日志的实时读取、正则关键词搜索过滤与一键清理。

## 🚀 核心技术特性与优势

我们致力于在本地硬件环境中构建起一座连接“大模型智慧”与“日常即时通讯协作工具”之间的桥梁。Nexora Agent 凝聚了多项硬核本地系统工程设计：

### 1. 🟢 零配置本地绿色沙箱 (.node-sandbox)
- **无感环境**：安装包中内置了一套完整、绿色的 Node.js 沙箱运行环境。即使用户电脑从未安装过 Node.js，也能做到“双击即用”。
- **自适应降级探活**：如果内置 Node 沙箱在某些受限系统（如缺失 VC++ 运行库或受 Windows 组策略限制）中无法被调起，系统会自动触发降级探活机制，检测并适配系统全局的环境变量 Node 路径，确保软件的最大启动兼容性。

### 2. 🔒 数据隐私本地化与长期记忆中枢 (MEMORY.md)
- **完全自主受控**：所有的聊天记录、大模型 API 密钥以及会话凭证统统只保存在您本机的 `%USERPROFILE%\.openclaw` 目录下。没有一字节的个人隐私会被上传到第三方服务器。
- **本地长期记忆栈**：内置基于本地 Markdown 格式的增量记忆同步机制。AI 助手会在对话结束时，自动把用户的偏好和事实沉淀写入本地记忆文件，并在下次对话启动时自动作为 Prompt 上下文拼装唤醒，越聊越懂你。

### 3. 🛡️ 双通道网络代理加速与通讯风控直连隔离
- **内置 Mihomo 加速内核**：内置高性能 Mihomo (Clash Meta) 二进制内核，自动托管并代理大模型 API 访问流量，彻底告别国内访问大模型端点连接超时、Key 被封禁的问题。
- **防封风控隔离白名单**：微信、飞书等渠道的连接若经过海外代理，会被腾讯和字节服务器瞬间触发风控（异地登录封号）。Nexora Agent 自研了“进程级环境变量按需注入”与域名直连规则，强制将微信、飞书, QQ, 钉钉的所有心跳与数据连接剥离出代理隧道直连公网，从根本上隔离了安全隐患，保障微信号和资产安全。

### 4. 🗣️ 本地全离线神经网络语音运行时 (ASR/TTS)
- **Sherpa-Onnx 推理引擎**：采用新一代 Kaldi 语音引擎与 ONNX Runtime，无需 Python 深度学习框架即可在本地高效运行。
- **离线中英混合发音**：支持下载高品質本地 VITS 神经网络语音包，提供极具人情味、自然流畅的中英双语朗读。
- **局域网语音桥接 API**：本地 18791 端口自带 HTTP 网桥，允许独立运行的通讯通道机器人通过 API 直接向本机的语音引擎投递文字，并由本机的扬声器朗读播放（支持 FIFO 单队列控制与一键静音）。

### 5. 🤖 物理级屏幕控制与桌面模拟自动化 (Computer Use)
- **open-computer-use 框架**：整合本地自动化控制接口，让大模型可以通过视觉模态实现“物理级”操作用户的电脑屏幕。
- **物理接口驱动**：调用本地自适应 DPI 截图脚本截取当前屏幕，并使用 Win32 API（SetCursorPos / mouse_event）向操作系统发送最底层的鼠标位移、物理点击与键盘输入指令。

### 6. 📊 可视化伪终端与彩色日志渲染
- **内置 Pty 伪终端**：通过 C++ 原生模块编译的 `node-pty` 守护后台网关子进程。
- **xterm.js 实时渲染**：将子进程的标准输出（stdout）中的彩色 ANSI 控制字符，完美以终端界面的形式呈现在前端，既充满工程美感，又极易排错定位。

## ⚙️ 技术架构简述

Nexora Agent 采用解耦的多进程拓扑架构设计，分为 GUI 交互层（Electron）与本地微服务网关层（OpenClaw 调度中枢 + 辅助运行时）：

```
                    +------------------------------------+
                    |        Electron 主进程 (Node)      |
                    +-------------------+----------------+
                                        |
                 +----------------------+----------------------+
                 | (child_process.fork) | (child_process.spawn)| (child_process.spawn)
                 |                      |                      |
+----------------v----------------+  +--v-----------------+  +-v------------------+
|    OpenClaw 本地网关 (中枢)     |  | Mihomo 代理加速内核|  | Voice Runtime 语音|
+----------------+----------------+  +--------------------+  +--------------------+
                 |
        +--------+--------+
        | (Plugin Loaders)|
        | - 微信通道插件   |
        | - 飞书通道插件   |
        | - QQ 机器人插件  |
        +-----------------+
```

## 👣 极简使用流程

> 详细配置说明与图文教程请参阅：[新手快速入门指南](docs/getting-started.md)

1. **第一步：安装与启动**
   - 下载最新发布的桌面安装包并双击安装。
   - 打开软件，点击左上角 **「启动 Nexora Agent」** 按钮，等待指示灯由红变黄，最终变为**🟢 绿色（已就绪）**。
2. **第二步：配置 AI 的大脑**
   - 进入左侧「模型配置」菜单，填入您的 API Key（如 DeepSeek 等平台 Key），然后保存。
   - 在「模型会话」中发送 `你好` 进行快速通路测试。
3. **第三步：渠道绑定**
   - 进入「通讯管理」菜单，在对应卡片（如微信）点击 **「扫码绑定」**，掏出手机扫码确认授权登录。
   - 当卡片状态变为“已绑定”后，即可拿另外一部手机发送消息测试自动回复！

## 🛡️ 陌生人安全授权与白名单

为了防止陌生人在群聊中刷爆您的 API Token 额度，Nexora Agent 预设了前置安全防火墙：
- 未配对的用户向您的 AI 助手发消息时，助手不会调用大模型回复，而是返回一个提示及 **4位配对码**。
- 您在电脑客户端的「通讯管理」中审核该配对申请并点击「同意」后，对方才可与您的 AI 进行正常聊天。
- 您亦可在配置中关闭此功能以实现全局无限制开放。

## 👨‍💻 开发者说明

如果您是开发者，想要基于本项目进行二次开发或自己尝试打包发布：

```bash
# 1. 克隆代码仓库
git clone https://github.com/2014-y/ClawAI.git
cd Nexora-Agent

# 2. 安装项目依赖 (自动拉取所需的通道与核心包)
npm install

# 3. 运行本地开发环境进行调试
npm run app:start

# 4. 一键编译打包出 Windows 下的绿色免安装目录与 NSIS Setup 安装包
npm run app:dist
```

> **注意**：打包时，本项目会自动执行差异化 ASAR 归档（`asarUnpack` 策略），将 C++ 二进制 DLL 扩展和 EXE 辅助程序释放在 unpacked 目录下，确保操作系统能正确调用底层 Native 接口。所有的打包输出文件都会存放在根目录的 `dist/` 文件夹下。
